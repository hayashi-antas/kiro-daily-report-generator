# Daily Report Generator

ルールベースで日報文を生成する、単一HTML構成のシンプルなWebアプリです。  
Kiroを使った仕様駆動開発（Spec-Driven Development）の体験を目的として作成しました。



## 概要

このアプリは、箇条書きで入力した「今日やったこと」などをもとに、  
**上司に提出できる丁寧な日本語の日報文（です・ます調）**を生成します。

- 外部API・LLM不使用
- ルールベースの文字列整形
- オフライン動作
- 単一HTMLファイルで完結



## 主な機能

- 箇条書き入力の正規化（記号除去・空行除外）
- 丁寧語（です・ます調）の日報文生成
- 学び・明日の予定セクションの自動付与
- 生成結果の編集・文字数カウント
- クリップボードコピー（フォールバック対応）
- 入力内容の localStorage 自動保存



## 使い方

1. `index.html` をブラウザで開く  
2. 「今日やったこと」を箇条書きで入力  
3. 必要に応じて「学び」「明日の予定」を入力  
4. 「日報を生成する」をクリック  
5. 内容を確認・微調整してコピー

※ ビルドやサーバー起動は不要です。



## 技術スタック

- HTML5
- JavaScript（Vanilla）
- Tailwind CSS（CDN）
- Browser APIs（localStorage / clipboard）



## 設計方針

- 仕様駆動開発（Spec-Driven Development）
- 仕様 → 設計 → 実装 → 検証の流れを重視
- 日本語生成はルールベースに限定し、自然さよりも仕様の明確さを優先
- 修正

## 変更・改修に関する方針

- 本プロジェクトでは、**コードを直接修正することを前提としない**
- 挙動の変更や機能追加を行う場合は、必ず先に「仕様書」を作成する
- 仕様書に基づき、実装作業は Agent（AI または自動化ツール）に委ねる
- 人は「何を変えたいか」「どう振る舞うべきか」という仕様定義に集中する

この方針により、  
実装よりも **設計・思考プロセスの明確化** を重視した開発体験を目的としている。

## 制約事項

- 文脈理解や高度な自然言語生成は行いません
- 日本語の自然さには一定の限界があります（LLM未使用のため）



## ライセンス

個人学習・研修用途を想定しています。
